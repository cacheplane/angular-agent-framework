# AG-UI a2ui Example (PR-B2) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the `ag-ui/a2ui` cockpit example — a port of the `chat/a2ui` flight-booking A2UI flow onto the AG-UI runtime, completing the AG-UI generative-UI trio (tool-views, json-render, a2ui).

**Architecture:** The agent emits an A2UI envelope (`---a2ui_JSON---`-prefixed JSONL: dataModelUpdate / surfaceUpdate / beginRendering) as the AI message's content. Over AG-UI that content streams via `TEXT_MESSAGE_CONTENT` (the reducer accumulates it verbatim), the chat lib's content classifier detects the prefix and mounts `<a2ui-surface>` against the stock `a2uiBasicCatalog()`. Purely example work — no chat-lib or adapter change (the `customEvents` signal that would enable *live* progressive a2ui already landed on main via #606, but this graph delivers the final surface in content, which renders over AG-UI today).

**Tech Stack:** Angular 20 (standalone, signals), `@threadplane/chat` (`a2uiBasicCatalog`, `A2uiSurfaceComponent`), `@threadplane/ag-ui`, Python LangGraph + `ag-ui-langgraph` (FastAPI/uvicorn/SSE), Nx, Playwright + aimock fixtures.

**Branch:** `claude/ag-ui-a2ui`. **Spec:** `docs/superpowers/specs/2026-06-06-ag-ui-spec-examples-design.md` (PR-B2 section).

**Constraint:** Make NO reference or mention of the two external projects flagged during brainstorming in code, docs, comments, or commits. Per the repo's standalone-examples rule, duplicate the graph code into this example; never import across examples.

**Ports:** angular 4324, langgraph (backend) 5324.

## Port source & key facts
- Source: `cockpit/chat/a2ui/` (python graph + angular app + e2e fixtures).
- The graph (`StateGraph(MessagesState)`: `route → build_form / search_flights / confirm_booking → generate_title → END`) emits the A2UI surface in AI message content via `_wrap_envelopes` (structured output + `_emit_with_retry`). NO custom events, NO graph state — so no lib change, no state-sync needed.
- The chat graph compiles with `_builder.compile()` (no checkpointer) because it runs on the LangGraph platform. **ag-ui-langgraph requires a checkpointer** — add `MemorySaver` (as json-render/tool-views/interrupts/streaming all do).
- `chat/a2ui` has NO `server.py` (it runs via `langgraph.json` on the platform). Create a new `server.py` for the ag-ui-langgraph/uvicorn runtime (mirror `cockpit/ag-ui/json-render/python/src/server.py`).
- `generate_title` calls the LangGraph SDK against `LANGGRAPH_API_URL` (absent under ag-ui-langgraph) and already swallows errors — graceful no-op. Keep it.
- aimock mocks the OpenAI provider below the transport, so the `chat/a2ui` fixtures (structured-output `BookingFormSpec`/etc. calls) port transport-agnostically.

  template: `
    <example-chat-layout>

        </div>
      </chat>

})
export class A2uiComponent {

  protected readonly catalog = a2uiBasicCatalog();
  protected readonly suggestions = WELCOME_SUGGESTIONS;
  protected send(text: string): void { void this.agent.submit({ message: text }); }
